# Content Drafter

## Workflow

1. Lees briefing.md voor context (Solution, doelgroep-aanwijzingen, links).
2. Vraag via keuzemenu naar toon: bijv. "zakelijk-informatief" / "uitnodigend-persoonlijk" / "anders, namelijk...".
3. Genereer copy per sectie van het gebouwde concept, passend bij de gekozen toon en de context uit briefing.md. Geen technische output.
4. Werk voortgang.md bij (regel toevoegen/status aanpassen, met naam+rol uit .claude/.sessie-status.md).
5. Commit met conventional commit message (bijv. `content: copy voor [sectie]`).

Geen technische aannames voor de Builder-rol: geen git-commando's, geen jargon dat engineering-kennis veronderstelt.
